Frag Arena — Music Prompt Kit // UT99-style

Copy/paste prompts for Suno/Udio, one per game state, modeled on how Unreal Tournament actually did it. Tap any block to copy. The combat family shares 140 BPM / E minor so tracks crossfade and layer cleanly.

The one rule that makes everything work: lock every combat-family clip (combat, low-health, overtime) to the same BPM and key (140 BPM, E minor). That's what lets you crossfade between intensity tiers and stack the low-health layer on top of the running combat bed without clashing — the same trick Doom 2016 used. Menu / warmup / stingers can float free.

How UT Switched Music (the model to copy)

UT played one tracker module per level, internally divided into numbered song sections (Section 0 ≈ ambient, 1 ≈ combat, 2 ≈ intro). The engine just said "jump to section N with transition T"; the module's own loop kept it seamless. Copy the transition enum 1:1:

TransitionBehaviorUse for
MTRAN_Instanthard switch (~0.33s micro-fade)combat hits NOW — low-health, overtime start
MTRAN_Segueswitch at next bar, no volume fademusical section change, same track
MTRAN_FastFade~1.0s crossfadequick but non-jarring swap
MTRAN_Fade~3.0s crossfadestate → state (default)
MTRAN_SlowFade~5.0s crossfadecombat → calm cooldown, mood shift
Key gotcha UT left to you: the engine had no automatic "enemy nearby → combat music" detector — mappers wired it by hand. Build the policy layer yourself: EXPLORE → COMBAT on damage/enemy-proximity (MTRAN_Instant), with a 5–8s cooldown/hysteresis before dropping back (MTRAN_SlowFade) so it doesn't flip-flop. Keep music state client-local, driven by each client's own view of combat.

WebAudio — the 80/20 for Frag Arena

  • Ship the UT model first: one combat loop + menu theme + victory/defeat stingers. That alone delivers the arena feel.
  • Gapless loops: AudioBufferSourceNode with loop=true + trimmed loopStart/loopEnd. Re-firing <audio> leaves audible gaps — don't.
  • Crossfade states: each state gets its own permanent GainNode into a music bus; ramp out/in with an equal-power curve (setValueCurveAtTime) to avoid the mid-fade volume dip. 1.5–3s for state changes, ~0.3s for combat↔low-health.
  • Low-health = vertical layer: start the overlay stem at gain 0, phase-locked to the combat bed (same ctx.currentTime, same BPM+key), fade it in. Biggest "modern feel" win for least effort.
  • Duck music under SFX: separate music/SFX gains; on loud events setTargetAtTime(0.35, t, 0.05) then restore. In a competitive FPS, footsteps/reloads are gameplay info — never let music mask them; keep the combat bed low/mid-heavy.
  • Clock hygiene: schedule against ctx.currentTime, not setTimeout. Call ctx.resume() inside the first user gesture (the Play/Enter click); preload + decode all buffers up front.

Generation Tips

  • Udio tends to win on clean instrumentals + section regeneration (good for extracting a loop); use Suno for the menu hero theme. Generate 3–4 takes per prompt — they vary a lot.
  • These tools don't output true seamless loops. Always trim to a clean bar boundary at a zero-crossing in Audacity/a DAW, and crossfade the seam if it clicks. Budget this step per clip.
  • Consistency for crossfade: put the exact 140 BPM, key of E minor in every combat-family prompt. Non-negotiable for layering.
  • Stems for layering: either regenerate variants at fixed BPM/key, or run one combat take through a stem splitter (Demucs) to get drums/bass/lead as vertical layers.
  • Always add instrumental / no vocals and, for loops, no build, no drop, steady energy.
