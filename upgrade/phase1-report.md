## Workaround applied (deviation from plan)
The plan's R9 anticipated a `ws`/`@clusterws/cws` server-transport leak (net/tls/http). **That did NOT occur.** Instead the FIRST `vite build` failed because nengi's *browser client* (`nengi/core/client/Client.js:12`) does `import { EventEmitter } from 'events'` — a Node built-in that webpack 4 auto-polyfilled but Vite/Rollup externalizes → `"EventEmitter" is not exported by __vite-browser-external`. nengi's client genuinely needs a real EventEmitter, so an empty stub is wrong. **Fix:** `npm i events@^3` (pure-JS Node `events` polyfill) + `resolve.alias.events` → `node_modules/events/events.js` in vite.config.js. Build then succeeds. No `ws`/`@clusterws/cws` alias was needed (server transport did not leak into the client graph).
